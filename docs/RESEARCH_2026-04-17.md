## What It Is

RefLab is a professional learning and training platform for football referees.
The MVP focuses on one core learning loop: **Tests → Feedback → Dashboard insights**.

Target users: Football referees who want to improve decision-making, rule knowledge,
and match performance.

**Language:** English  
**Stack:** React + TypeScript + Vite + Tailwind (frontend) | Supabase (auth, DB, storage) | Edge Functions (planned)

---

## Backend — Supabase Schema (36 migrations)

Rich schema with multiple features. Tables identified from migrations:

### Auth & Profiles
- `auth.users` (Supabase managed)
- `profiles` — username, avatar_url, bio, xp, level, rank, tier, trust_score, etc.

### Tests & Learning
- `tests` — slug, title, is_active
- `test_questions` — test_id, order_index, question_text, options A-D, correct_option
- `test_attempts` — user_id, test_id, status (in_progress/submitted), started_at, submitted_at, score_correct, score_total, score_percent, answers JSON, explanation_cache

### Billing (Stripe)
- `stripe_customers` — user_id, stripe_customer_id
- `stripe_subscriptions` — user_id, subscription_id, price_id, plan (pro/plus), status, current_period_end
- `stripe_webhook_events` — event_id, type, created, processed

### Social
- `posts` — user_id, content, media_type (text/image/video/audio), media_url, original_post_id (repost), like_count, comment_count, repost_count, save_count
- Enums: `post_media_type`
- RPCs: get_post_by_id, profile_feed, self_profile_feed

### Notifications
- `notifications` — user_id, type, title, message, read, actor_id, reference_id

### Messages
- `messages` — conversation-based DMs with realtime

### Settings
- `settings_tables` — user settings

### Video
- `video_decision_tables` — video scenarios for referee decisions
- `video_action_sanction` — action/sanction mappings

### Search
- `search_users_rpc` — user search with trigram improvement
